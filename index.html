Git e seus comandos básicos e avançados

O que é o GIT?

GIT é uma sistema de controle de versões distribuídos ou DVCS. Estes sistemas de controle possuem a função de registrar quaisquer alterações feitas em cima de um código, armazenando essas informações e permitindo que, caso seja necessário, um(a) programador(a) possa regredir a versões anteriores de uma aplicação de modo simples e rápido.

Para facilitar o uso e entendimento dessa ferramenta, abaixo segue uma lista das principais coisas que você precisa saber para poder utilizar dessa ferramenta:

  • Repositório: São espaços criados para armazenar seus códigos
  • Branch: É o nome dado a uma versão/ramificação do projeto
  • Merge: Comando responsável por unir modificações para a branch principal

Principais comandos do GIT

Agora que conhecemos alguns termos a respeito do GIT, que tal aprendermos seus comandos mais básicos?

  • Init: Esse comando cria um repositório novo, seja ele remoto ou local, ou reinicializa um repositório já existente;
  • Clone: Como o nome sugere, este comando clona o código do repositório para manipulação em outro ambiente;
  • Commit: Este comando move os arquivos da State Area para um repositório local;
  • Add: Já este comando, como sugere seu nome, adiciona um arquivo para a chamada Staging Area, ou seja, o prepara para um Commit;
  • Push: Envia os arquivos do repositório local para o remoto;
  • Pull: Este faz o contrário do Push, ele move arquivos de repositórios remotos para locais;
  • Merge: Serve para unir arquivos alterados ao arquivo original, básicamente unir os commits com a branch principal, na maioria dos casos;
  • Log: Permite ver o histórico de commits de um arquivo ou usuário;
